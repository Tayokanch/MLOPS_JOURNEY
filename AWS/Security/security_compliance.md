# Security and Compliance

## What is AWS Compliance

- AWS omplaince and regulatory framworks are sets of guidelines and best practies that AWS must follow or Meet. 

## How To Access AWS Reports

`AWS Artifact`
- AWS undergoes **audits** and **verification reviews** by various governing bodies to ensure that they are compliant with a variety of different regulatory frameworks. Reports of  these **audits** can be downloaded from a Service called `AWS Artifact`

`Customer Compliance Center`

- This is a **central location website** where we can reseach for cloud-rlated regulatory requirements and how they impact our industry. On this Website we can search for the following:
  - Indentify Regulatory Requirements
  - Browse Country-Specific laws/requirements
  - Discover how companies in various industries solved compliance, governance challenges
  - AWS answers to key compliance questions
  - Auditind and Security checklist
  - Reference Architectures with best practices

`AWS Audit Manager`

- When we deploy resources to the cloud and our organization has to be compliant with different regulatory frameworks, we would need to make sure that all of the services and resources also meet those regulatory requirements.**An AWS Audit Manager continuously** collects data to prepare for auits and ensures that you comply with regulatory standards.


`AWS Config`
- This tracks how the resource is configured and records previous configuration state, so we can see how the configs have changed over time
