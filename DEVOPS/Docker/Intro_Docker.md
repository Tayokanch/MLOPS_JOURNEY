# Docker Overview

- `Docker` helps in running a service in a seperate container with its own dependencies and its own libraries, all on the same VM and the OS, but within a seperate enviornments or Container. The main purpose of `Docker` is to package and containerize application, and to sheip them, and to run them anyhwere, anytime, as many times as you want.

- It solves OS compatability issues for different services.
- it solves long setup time
- It is used to run services on a different environment.


# CONTAINER
- `Containers`: These are completely isolated environments, as in, they can have their own processes or cervices, their own network interfaces, their own mounts, just like virtual machies, expect they all share same OS kernel.


# DOCKER IMAGE
- `IMAGE`: An Image is a package or template, just like a VM template that you might have worked with in the Virtualizaton world, its used to created one or more containers. Containers are running instances of images that are isolated and have their own environments and set of processes
